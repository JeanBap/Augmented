User Sign-in Flow

User visits any subdomain
->
Clicks "Sign in"
->
Google OAuth or Magic Link
->
Supabase sets cookie on .raisereadybook.com
Cookie shared across all subdomains
->
Logged in everywhere instantly
->
Dashboard shows purchases + saved work from all apps

Purchase Flow (Books/Services/Exports)

User clicks "Buy" on any app
->
API creates Stripe Checkout session
->
Stripe hosted checkout
Payment succeeds
->
Stripe webhook fires
->
Purchase saved to Supabase
->
User sees it in dashboard

Migration Phases (8-10 days)

1
Foundation - Create repo, install deps, configure Supabase auth for subdomains, set up Stripe products
2
Content migration - Move blog/book/service content from Augmented + raisereadybook.com into apps/www
3
Personal finance tools - Port 13 calculators from calculateapr into apps/app, converting React Router to Next.js App Router
4
Financial Model Pro - Port engine + editor components from calculateapr into apps/finance
5
Deploy to Vercel - Create 3 Vercel projects, connect to GitHub, set env vars
6
DNS - Add CNAME records in Namecheap for app. and finance. subdomains
7
Decommission - Cancel Lovable, remove Netlify, archive old repos, set up redirects
8
Verify - Test cross-subdomain auth, payment flows, exports, all calculators
